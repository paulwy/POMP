# 系统启动自动化功能

本文档描述企业管理系统在启动时的自动化检测和初始化功能。

## 🚀 启动流程概览

系统启动时会自动执行以下步骤：

```text
1. 加载环境变量
2. 初始化日志系统
3. 连接并检测数据库
4. 连接并检测Redis
5. 初始化服务组件
6. 构建应用状态
7. 检测Ollama服务 (AI功能)
8. 初始化默认数据
9. 启动HTTP服务器
```

## 📊 启动时自动化检测

### 1. 数据库检测

- ✅ 建立数据库连接池
- ✅ 执行连接测试 (SELECT 1)
- ✅ 记录连接状态

### 2. Redis检测

- ✅ 连接Redis服务器
- ✅ 发送PING命令验证
- ✅ 记录连接状态

### 3. Ollama AI服务检测

- ✅ 检测Ollama服务是否运行
- ✅ 列出可用模型
- ✅ 如果未连接，提供友好提示并降级到本地知识库

### 4. 默认数据初始化

- ✅ 检查是否存在默认管理员
- ✅ 如果不存在，自动创建默认账户
  - 用户名: `admin`
  - 密码: `admin123`
  - 名称: `系统管理员`

## 🔧 健康状态说明

| 状态 | 说明 |
|------|------|
| `healthy` | 所有核心服务正常运行 |
| `degraded` | 部分服务不可用，系统仍可运行 |
| `unhealthy` | 核心服务不可用，系统无法正常运行 |

## 🔍 监控建议

### 使用健康检查进行监控

可以通过监控详细健康检查API来监控系统状态：

```bash
# 检查系统状态
curl http://localhost:3000/api/health/detailed

# 或使用jq解析状态
curl -s http://localhost:3000/api/health/detailed | jq '.status'
```

### 服务状态含义

- **database**: 数据库连接状态
- **redis**: 缓存服务状态
- **ollama**: AI辅助服务状态 (可选)

## ⚠️ 注意事项

1. **安全提示**
   - 默认管理员密码仅供开发使用
   - 生产环境请立即修改默认密码
   - 删除或禁用默认管理员账户

2. **Ollama连接**
   - Ollama是可选服务，不影响核心功能
   - 未连接时自动使用本地知识库
   - 可随时启动Ollama增强AI功能

3. **数据备份**
   - 定期备份数据库
   - 记录自定义配置变更
   - 保存重要的业务数据

4. **资源监控**
   - 监控数据库连接数
   - 监控Redis内存使用
   - 监控系统日志
